Workout Timer

The Concept: Make workouts, select exercises, change reps, time yourself, and get fit!

An app to help users design workouts with fine detail and then time their reps, laps, sessions - all at once! The app allows for customizing the number of reps, rest time, and more.

The Project: React-Redux Frontend, a Restful API made with Hapi.js(express.js adjacent) on backend, and a PostgreSQL database

I created this app to show off my Node.js abilities. The infinite carousel makes use of some pretty awesome state management and clever code to keep it lightweight, adaptable, and flexible. The app posts database changes to user's workouts in real time for a really nice user experience. The backend makes use of features like tokenization and request schema to keep data secure.

Best Practices All Around

I am a stickler for well-organized code! I want anyone looking at my work to know exactly what their looking at and how it works. Solid naming conventions, good commenting practices, and clear seperation of concerns make every block of code pleasant to work with - and easy to modify!

Everything in the app is coded with functional design in mind for maximum scalability. The whole thing is coded that exact same as you would for an enterprise-level application - well organized and single-service components, and super nice readability.

Timer, CountDown, Pause, Lap - Oh My!

The timer is a fun example of implimenting a complex component with multiple functionalities. The timer can be stopped, started, and paused - it can also be used for rest countdowns and tracking smaller timeframes within a larger one. This allows for timing individual groups of reps or rounds while also timing the whole workout.

Finally, all of these datapoints are recorded and stored away in a database for later displaying in the "view times" section. In the future I would love to make use of some special graph-rendering techniques (which I learned from making a stock indicator's analysis tool) to display the users fitness progress!
